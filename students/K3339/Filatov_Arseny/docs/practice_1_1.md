# Практика 1.1 — Ledger Lite (in-memory)

## Назначение

Первая итерация сервиса учёта личных финансов: **без постоянной базы данных**. Данные хранятся в памяти процесса, после остановки сервера списки сбрасываются.

Задача этапа — освоить **FastAPI**, модели **Pydantic v2**, полный **CRUD** и вложенные объекты в JSON (кошелёк и категория внутри транзакции, список тегов).

## Предметная область (модели API)

- **Wallet** — кошелёк: `label`, `currency_code` (например `RUB`).
- **Category** — категория расходов/доходов и опциональный лимит `spending_cap`.
- **Tag** / **TagCreate** — теги; при создании транзакции задаются как новые имена, сервер присваивает `id`.
- **Transaction** — операция: `memo`, `value`, `kind` (`income` | `expense`), дата `booked_on`, вложенные `wallet`, `category`, список `tags`, опционально `note`.

При **POST/PUT транзакции** во вложенных `wallet` и `category` нужно передать **существующие** `id` из текущих списков кошельков и категорий.

## Установка и запуск

Из корня репозитория перейдите в папку практики, активируйте виртуальное окружение (например общее `Lr1/.venv`) и установите зависимости:

```bash
cd Lr1/practice_1_1
pip install -r requirements.txt
uvicorn app.main:app --reload
```

По умолчанию API доступно по адресу `http://127.0.0.1:8000`.

**Документация OpenAPI (Swagger UI):** `http://127.0.0.1:8000/docs`  

Переменные окружения и файл `.env` для этой практики **не требуются**.

## HTTP API (кратко)

| Метод | Ресурс | Описание |
|:-----:|--------|----------|
| GET | `/` | Приветственная строка |
| GET, POST | `/wallets` | Список / создание кошелька |
| GET, PUT, DELETE | `/wallets/{id}` | Чтение / замена / удаление |
| GET, POST | `/categories` | Список / создание категории |
| GET, PUT, DELETE | `/categories/{id}` | Чтение / замена / удаление |
| GET, POST | `/transactions` | Список / создание транзакции |
| GET, PUT, DELETE | `/transactions/{id}` | Чтение / замена / удаление |

Удаление категории или кошелька запрещено, если на них ссылаются транзакции.
